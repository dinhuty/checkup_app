Basic design of the same-day operation system

Requirements Definition Section

Background section

Summary section

This requirement definition defines requirements for the following mechanisms:

  • Setting a schedule for operation on the day
  • Clinical management and health check results registration during the same day operation
  • Master settings required for the above

Function overview

It will be separated from CHECKUP and will be developed as an operating service on the day. Operating on the day is mainly a function for round-trip health checkups, but it can also be used as a facility health checkup.

Regarding items managed by Redis etc. of information not covered in the same day operation table

  • overview

    Items that exist in CHECKUP's DB are obtained through the CHECKUP API and applied to the view on the screen. However, since high-frequency ratings of APIs should be avoided, depending on the item, we will consider whether to manage them in the database of the same day or in sessions.

  • Consider

    Since I am currently using Redis for CHECKUP and other sites, I am considering using Redis. If Redis is difficult due to operational reasons, we also consider Dynamodb from AWS.
